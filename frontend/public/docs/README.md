# Public docs

PDFs and other static documents served at `/docs/<filename>` by the
frontend.

## Contents
- `ForgeSlicer-Texture-Tutorial.pdf` — in-depth (10-page) tutorial for
  the Texture Library feature. Generated by
  `/app/scripts/build_texture_tutorial.py`. Linked from the Texture
  Library dialog header ("Tutorial PDF" link).

## Regenerating

```bash
cd /app && python3 scripts/build_texture_tutorial.py
```

The build script is idempotent — it re-renders the schematic
pattern thumbnails first (via `scripts/render_texture_thumbs.py`),
then composes the PDF in-place. After regeneration, refresh the
browser; the new PDF is served immediately (no rebuild needed).

## Adding a new tutorial

1. Drop a new `build_<topic>_tutorial.py` next to the texture script.
2. Output to `/app/frontend/public/docs/ForgeSlicer-<Topic>-Tutorial.pdf`.
3. Link it from the relevant dialog/popover (see `TextureLibraryDialog.jsx`
   for the pattern).
